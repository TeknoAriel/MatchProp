# Archivo de documentación

Material **histórico**: sprints cerrados, planes de cierre, auditorías extensas y guías reemplazadas.  
**No** usar como fuente de verdad para el producto actual; ver [FOCO_2026.md](../FOCO_2026.md), [masterplan.md](../masterplan.md) y [ALINEACION_MASTERPLAN.md](../ALINEACION_MASTERPLAN.md).

## Contenido (referencia)

| Tipo                  | Archivos (ejemplos)                                                                                    |
| --------------------- | ------------------------------------------------------------------------------------------------------ |
| Sprints               | `matchprop-sprint1.md`, `matchprop-sprint2.md`, `sprint-*.md`                                          |
| Planes / cierres      | `plan-completamiento-100.md`, `PLAN_DE_TRABAJO_CIERRE_VERSION.md`, `SPRINT_SIGUIENTE_100_OPERATIVO.md` |
| Auditorías / informes | `AUDIT_MATCHPROP.md`, `AUDITORIA_FULLSTACK.md`, `INFORME_AUDITORIA_PM*.md`, `AUDITORIA_V1_DEPLOY.md`   |
| Guías / estado viejos | `GUIA_IMPLEMENTACION_V1.md`, `GUIA_DEMO_V2.md`, `ESTADO_MATCHPROP_V2.md`, `SETUP_V2.md`                |

Los enlaces **entre archivos dentro de `archive/`** siguen siendo relativos al mismo directorio.  
Para enlazar documentos **canónicos** que viven en `docs/` (ej. `masterplan.md`), usar ruta **`../archivo.md`** desde un `.md` dentro de `archive/`.
